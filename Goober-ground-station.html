Introduction

Goober Ground Station tracks and receives data from satellites, later decoding data from them to get useful information and interesting information. The station is funded mostly by me,

Software and Hardware Tables

Goober Ground Station (GGS), heavily relies on third party software and hardware. The following tables lists all software and hardware used in the ground station.

Software

Name Link Picture Description
SDR++ https://www.sdrpp.org SDR++ is a cross platform SDR software used for recording and listing to the audio, also used from replaying baseband files for things like SSTV. I all so use it on my RaspberryPi 5 as a server, so I can connect to the SDR over WiFi useing 'SDR++ Server' and 'RTL-TCP' (RTL SDR's only).
SDR# https://airspy.com/download/ SDR# is a SDR software similar to SDR++, but has downloadable plugins and many other built-in plugins.
SatDump https://www.satdump.org/download/ Used for satellite decoding/recoding; NOAA APT, NOAA DSB, METEOR M2-x LRPT 72k. Also be automated to record and track the satellites.
RX-SSTV https://www.qsl.net/on6mu/rxsstv.htm RX-SSTV is a program used for decoding SSTV over a virtual audio cable.
MMSSTV https://hamsoft.ca/pages/mmsstv.php MMSSTV is a program used for decoding SSTV over a virtual audio cable.
WSJT-X https://wsjt.sourceforge.io/wsjtx.html
VBCable driver https://vb-audio.com/Cable/ VBcable is a program used to take the audio from SDR++ and outputs it in a diffrent audio output, so i can use the output as an input to other programs that can decode things like SSTV or FT8.
Orbitron https://www.stoff.pl/downloads.php Orbitron is over 20 year old, but it still work just fine. Its used to track and the satellite passes over the station using TLE's found at celestrak.org.
